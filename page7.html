Synchonisation GitHub

La commande git remote add est utilisée pour ajouter un dépôt distant Git à votre référentiel Git local. La syntaxe complète de cette commande est git remote add , où est le nom que vous donnez au dépôt distant et est l'URL du dépôt distant que vous voulez ajouter.
git remote add origin <URL-du-dépôt-GitHub>
Pour récupérer l'URL d'un dépôt GitHub existant, vous pouvez suivre ces étapes :
  1. Ouvrez GitHub dans votre navigateur et connectez-vous à votre compte.
  2. Accédez au dépôt dont vous voulez récupérer l'URL.
  3. Cliquez sur le bouton vert "Code" en haut à droite du dépôt.
  4. Dans le menu déroulant, sélectionnez "HTTPS" ou "SSH", en fonction de la méthode que vous voulez utiliser pour cloner le dépôt.
  5. Cliquez sur le bouton "Copier" à côté de l'URL affichée pour copier l'URL dans votre presse-papiers.
La commande git remote -v est un moyen pratique d'afficher les informations sur les référentiels distants connectés à votre référentiel local, ce qui peut être utile pour suivre et gérer vos projets Git.
git push -u origin master
la commande git push -u origin master pousse la branche locale "master" vers la branche principale du référentiel distant nommé "origin". Elle définit également la branche distante comme branche de suivi pour la branche locale. Cette commande est généralement utilisée lors du premier push vers un nouveau référentiel distant, mais peut également être utilisée ultérieurement pour mettre à jour la branche principale du référentiel distant avec les modifications locales.
git push -u origin master
Une fois utilisée on peut utiliser la commande $ git push seul, il n’est plus nécessaire de spécifier la branche a synchroniser.
